## 2. kintone連携機能の改修報告

### 2.1 面談管理→契約管理 連携機能
- **新機能**：「契約レコード作成」ボタンを面談管理アプリに設置。
- ボタン押下により、契約管理アプリへ自動転送（レコード作成）可能。  
  → 人材名・案件名・番号など主要項目が自動反映。
- マッピング未反映項目（例：人材単価・契約単価）は、後続対応にて追加予定。

#### 🛠 現状の仕様
| 項目 | 現状 | 対応予定 |
|------|------|-----------|
| 案件名／案件番号 | 自動反映済み | ✅ 完了 |
| 人材名／人材番号 | 自動反映済み | ✅ 完了 |
| 人材単価・案件単価 | 未反映 | 🔧 佐藤が会議後に即日修正予定 |
| 稼働時間→金額自動算出 | 契約画面上で機能済み | ✅ |

> 面談管理側に単価フィールドが存在しないため、反映不可だった。  
> → 対応後、買掛・売掛テーブルの金額も自動反映される予定。

## 3. データ登録フロー全体整理

### 3.1 案件／人材／面談／契約の流れ
1. **案件アプリ**から「面談登録」ボタンで面談レコードを作成。  
   → 案件情報（番号・案件名など）が自動転送。
2. **人材アプリ**からも同様に「面談登録」可能。  
   → 対象人材情報が面談管理へ自動連携。
3. **面談管理アプリ**で「契約レコード作成」ボタンを押下。  
   → 契約管理アプリに必要情報を引き継いで新規作成。
4. **契約管理アプリ**内で請求金額・超過／控除時間などを自動算出。

### 3.2 契約作成の二経路
| 経路 | 概要 |
|------|------|
| 面談管理経由 | 面談情報をそのまま契約に変換（標準） |
| 契約管理直接追加 | 案件番号をルックアップで取得 → 自動補完される |

### 3.3 請求・支払管理
- 契約書・請求書宛先のみ手動入力が必要。  
- 他項目はすべて自動反映。  
- 将来的にはkintone内から請求書PDF自動生成まで拡張可能。

## 5. SEO・301リダイレクトの議論

### 5.1 鈴木氏からの提案
- 旧ProWorks（2019年版）から新ドメインへの移行に伴い、  
  **301リダイレクト**を設定してSEO評価を保持すべき。
- 理由：Googleの検索評価（ドメインパワー）がリセットされるリスク。
- 特に `/mypage`, `/pw` 等の旧URL構造を新URLに誘導する必要あり。

### 5.2 佐藤見解
- 技術的にはリダイレクト可能だが、旧画面構成と大幅に異なるため  
  ページマッピングの作業量が多い。  
  → 全URLではなく主要ページのみ対応する方が現実的。
- SEO影響は限定的だが、影響度を調査の上で判断する方針。

### 5.3 決定
- 鈴木氏：旧URL一覧とリダイレクト要否をまとめて共有予定。  
- 佐藤氏：社内マーケチームでSEO影響度を確認。  
- 双方で再検討後、必要な範囲でリダイレクト設定を実施。
