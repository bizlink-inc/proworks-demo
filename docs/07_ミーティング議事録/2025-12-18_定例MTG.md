## 2. プロワークス案件管理のUI/UX仕様調整

### アイコン・素材

- アイコン素材はフリー素材で統一
- 混在していたプロ版素材は入れ替え予定

### 入力レイアウト

| 項目 | 仕様 |
|------|------|
| 項目名 | 左寄せ |
| 入力欄 | 右寄せ |
| 必須/任意表示 | UI設計書に合わせて位置修正 |

### ヘッダー・メニュー

- ボディと幅調整（左／右寄せ）
- レスポンシブ時の動作方針を確認
- モバイル/タブレット時のメニュー表示はアイコン付き2行など仕様書に合わせ修正

### タグ・職種・スキル表示

| 項目 | 仕様 |
|------|------|
| タグ表示 | 最大3個（長文の場合は2個に） |
| 職種・スキル表示 | 2-3個・2行までで調整 |

### ボタン・バッジ

| 項目 | 仕様 |
|------|------|
| 「詳細を見る」ボタン | 位置・高さを揃える |
| バッジカラー | 「案件決定：赤」、他ステータスは基本枠なし、UI設計書に準拠しカラーコード指定 |
| 応募取消ボタン | 面談調整中以降は非表示等、要件を基本設計書で確認 |

### 検索欄・案件一覧

- レイアウト・枠設定もUI設計書に合わせて横幅・デザインを統一
- 応募済案件のない場合はスライダーでAIおすすめ案件を表示（UI設計書通り）

### デザイン細部

- カラー・フォント指定などデザイン細部は今後追加指示・リストで共有
- ディスプレイ環境やPDF化等で色味の違いが発生するため、カラー・フォントの細部指定もリストで改めて指示予定

## 4. Indeed連携・応募フロー設計について

### 現状のフロー

1. Kintoneに案件登録
2. XMLでIndeed投稿
3. 応募があれば人材DBにレコード作成

### 課題

- 自動人材DB登録時、フォームのデータが充分でない場合は追加情報収集が課題
- GoogleフォームかKintoneフォーム運用かは要調査
- ID付きフォームで情報紐付検討中

### 検討中の運用案

- サブDB（仮登録用）を経由し、充分な情報が得られた時点で本DB登録
- 運用複雑化・保守性も考慮し、改善案・自動化案を改めて検討し明日以降提案予定
