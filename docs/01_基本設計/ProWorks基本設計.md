## 2. 機能について

### 2.1 案件一覧

#### 案件表示について

1. 登録情報にマッチする案件を上位表示（AI とは別で実装）
2. AI マッチで該当したものは、① よりも上位表示させる
3. 担当者おすすめ表示。② よりも上位に表示（最優先）
4. 応募済み案件は案件一覧へ表示させないようにする
5. 案件の表示件数は 21 件毎に表示（3列×7行のグリッドレイアウト）

**おすすめ順の優先順位**：
- ③担当者おすすめ（最優先）
- ②AIマッチ
- ①登録情報マッチ（適合スコア）
- 同スコアの場合は新着順

#### 検索画面の項目

- **リモートチェックボックス**（複数選択可）
  - リモート、ハイブリッド、常駐
- **職種チェックボックス**（複数選択可）
  - 開発、インフラ、PM・PMO、コンサル、社内 SE、デザイン

#### 表示プルダウンについて

- おすすめ順、新着、開始時期、金額高い順、職種順

#### 案件応募のフローについて

- 応募ボタンをクリックした際にマイページの不足項目入力を促す
- ユーザ DB にフラグカラムを持たしたりして判定？

#### 表示タグについて

案件カードの左上に以下のバッジが表示されます（左から順）：

- **NEW**（赤色リボン）：新着案件（作成日時が14日以内）
- **担当者おすすめ**（オレンジ色リボン）：担当者おすすめフラグが設定されている案件
- **AIマッチ**（青色リボン）：AIマッチが実行済みの案件

これらのバッジは同時に表示される場合があり、左から「NEW → 担当者おすすめ → AIマッチ」の順に配置されます。

### 2.2 応募済み案件

#### 機能

- 応募した順で表示（1 番上が 1 番最近の案件）
- 応募中、面談調整中、案件決定、募集終了のステータス表示
- 面談調整中のステータスになった場合、応募取消ボタンは非活性
- 三ヶ月経過 → 応募済みの画面からは削除（Kintone では見れるようにする）
- 案件自体の応募が終了になったら応募終了と表示させるようにする
  - →Swift 案件 Kintone の案件 DB クローズ時。案件の詳細は表示しない。

#### 表示ステータス遷移

- 応募済み = 応募済み
- 面談調整中 = 面談調整中
- 予定決定 = 面談予定
- 案件参画 = 案件決定
- 見送り = 募集終了

### 2.4 通知機能

#### 新規登録のフロー

- 新規登録 → メール送信した旨を表示する画面 → リンクを踏んだ先マイページ

#### 通常フロー

- ログイン画面 → デフォルト画面（案件一覧）の予定

#### ユーザへ通知機能は下記を予定してます

| 通知種別             | アイコン | 説明                                                                                                                                                                                                      |
| -------------------- | -------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| おすすめ案件         | 🔔       | AI マッチ等でおすすめされた場合にユーザへ通知                                                                                                                                                             |
| 応募完了通知         | -        | 画面に表示                                                                                                                                                                                                |
| 面談予定             | 🔔       | 面談予定になったらユーザへ通知                                                                                                                                                                            |
| プロフィール記入通知 | 🔔       | 必須項目の記入がない場合（定期的に通知）                                                                                                                                                                  |
| 検索結果なし         | -        | 案件一覧 → 検索結果 → なしの時の画面「検索結果は見つかりませんでした。別のキーワードで試してください。検索ワードを短くするか、絞り込み条件を外すとヒットしやすくなります。0 件/全 ○○ 件で考えています。」 |

#### 各画面での通知詳細

##### 案件一覧 → 案件詳細 →「応募する」をクリックした後の画面

- 応募が完了しましたの通知をポップアップ表示し、ポップアップ内で応募済み案件確認ボタンを設置、案件一覧に戻るボタン設置で該当ページに戻る。

##### 応募済み案件 → なしの時の画面

- 応募済みの案件はございません。
- おすすめ案件をスライダー表示
- → 案件を探す or おすすめ案件を確認するで案件一覧ページへ

##### 応募済み案件 → ありの時の画面

- 案件一覧の案件と同様の表示のさせ方でステータスと応募取消ボタンを追加する。
- おすすめ案件についての通知の表示は相談させてください。
- 定型文と日時は追加。
  1. おすすめ案件の詳細を表示できるようにする
  2. 案件一覧へリンクを作成し表示できるようにする。

##### 面談予定になったらユーザへ通知

- リンクから応募済み案件ページを確認できるようにする。

##### プロフィール記入通知

- マイページ画面に飛べるようにする。

##### 全ユーザーへの通知

- サイトメンテナンスとか緊急の知らせなど

### 2.7 Indeed 連携

#### チャットワークへ連携

- Indeed 応募（ユーザ）→ フォーム送信 → フォーム回答 →DB 取込（検討中）
- Kintone で案件入力 →Indeed へ投稿連携（DB では実案件 or ダミー案件がわかるように設定する）

## 3. 応募に当たり必要最低限の情報

### 3.1 プロフィール

| 項目     | 機能定義                                                      |
| -------- | ------------------------------------------------------------- |
| 氏名     | 営業側から直接オススメ → 直接オファーを出したい人へ出せる機能 |
| フリガナ | 同上                                                          |
| 生年月日 | 同上                                                          |
| 電話番号 | AI マッチ                                                     |
| 郵便番号 | AI マッチ                                                     |
| 住所     | AI マッチ                                                     |

### 3.2 職歴・資格

| 項目                         | 機能定義  |
| ---------------------------- | --------- |
| 言語・ツールの経験           | AI マッチ |
| 主な実績・PR・職務経歴・資格 | AI マッチ |
| アップロード                 | AI マッチ |

### 3.3 希望条件

| 項目             | 機能定義                                                  |
| ---------------- | --------------------------------------------------------- |
| 希望勤務日数     | プログラムでマッチ → オススメとして登録情報から表示させる |
| 出社頻度         | 同上                                                      |
| 稼働可能時期     | 同上                                                      |
| 希望勤務スタイル | 同上                                                      |

### 3.4 希望単価

- プログラムでマッチ → オススメとして登録情報から表示させる

### 3.5 AI マッチのロジック

- **AI マッチ**: 経歴やスキルでマッチ。希望条件は含まない。
- **目的**: AI マッチでの抜けを防ぐ

#### ユーザ側

- 案件に対して AI マッチ実行
- そこで抽出されたユーザに AI マッチ案件として上位に表示
- 抽出されたユーザへは無条件に上位表示（誰に表示させるか admin 側で選択しない）

#### 営業側

- 案件に対して AI マッチ実行
- そこで抽出された人材を再度営業側で選定
- 選定後に評価が高い人へ直接連絡
